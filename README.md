<h3>Бот для приобретения инфо-продукта</h3>

<ul>
    <li>Команда start, запуск бота.<br>
        Приветствие, кнопка для обзора доступных продуктов"</li>
    <li>После нажатия на кнопку "Обзор продуктов", пользователь получает список доступных продуктов для приобретения.<br>
        Каждый элемент содержит обложку, описание и цену.<br>
        Под каждым продуктом есть кнопка "Купить продукт" с доп. указанием стоимости</li>
    <li>Кнопка "Купить продукт" переключает пользователя на YOOKASSA, где нужно оплатить указанную стоимость.</li>
    <li>После оплаты пользователь получает в бот чек о проведенном платеже от платежной системы.</li>
    <li>Пользователь получает после оплаты ссылку на скачивание видео.</li>
</ul>

<h4>Админка</h4>
<p>Администрирование доступно только для тех пользователей, чей user_id заранее добавлен в .env.
Система назначения/смены администраторов не реализована по желанию заказчика.</p>

<i>Доступный функционал:</i>
<ul>
    <li>Создание записи в БД для нового инфо-продукта</li>
    <li>Просмотр существующих продуктов, доступных для приобретения</li>
    <li>Для каждого продукта доступно архивирование или редактирование</li>
    <li>Архивированные продукты можно удалять без возможности восстановления</li>
    <li>Редактирование каждого параметра осуществляется посредством inline-кнопок</li>
    <li>Выгрузка пользователей бота csv-файлом</li>
    <li>Выгрузка покупателей продуктов csv-файлом</li>
    <li>Выгрузка истории изменений продуктов csv-файлом</li>
    <li>Выгрузка БД</li>
    <li>Выгрузка файла логов</li>
</ul>

<i>В репозитории не хранится база данных, лог-файл, .env, venv</i>
<i>В шаблоне .env лежит токен телеграм-бота, user_id администраторов, токен yookassa</i>
